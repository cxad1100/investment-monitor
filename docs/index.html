How the numbers are computed

Why not just use past returns? (the whole point)

Naive Markowitz feeds the optimizer each asset's average past return as its expected future return. This is its fatal flaw: a sample mean of returns is almost pure noise and just chases recent winners. The covariance matrix (how assets move together), by contrast, is statistically stable and genuinely useful. So this report uses covariance for everything and never uses trailing returns as a forecast.

Covariance & volatility (the trustworthy input)
  • Volatility = standard deviation of daily returns × √252 — how much an asset swings.
  • Covariance = how each pair moves together, estimated from 365 days of prices.
  • Portfolio volatility is not the average of asset volatilities — assets that zig while others zag cancel risk out. Exploiting that is the entire value of diversification.
Minimum Variance & Risk Parity (no return forecast)

Minimum Variance: scipy (SLSQP) finds the weights with the lowest possible portfolio volatility (long-only, ≤35% each, sum 100%). Needs only covariance.

Risk Parity: weights chosen so every asset contributes an equal share of total risk. Risk contribution of asset i = wᵢ × (Σw)ᵢ ⁄ portfolio variance. Also covariance-only — this is what risk-parity and minimum-volatility funds (e.g. the USMV ETF) actually do.

Black-Litterman (a return model that isn't momentum)

Instead of past returns, reverse the optimization: given the market's cap-weighted mix, what expected returns would make that mix optimal? Π = δ · Σ · w_market (δ = 2.5 risk aversion). These "implied" returns are the market's collective forecast. You can layer your own views on top (Black-Litterman blends them by confidence); with no views the report uses pure Π. Then the two BL portfolios run standard max-Sharpe / same-risk optimization on Π instead of on history.

Sharpe ratio & the risk-free rate

(portfolio return − 4.5%) ÷ volatility = return per unit of risk. The 4.5% cash rate is a fixed offset — it shifts every Sharpe equally, so it never changes the ranking of portfolios; it only sets the bar that "doing nothing" (holding cash) clears.

Simple ROI vs Money-Weighted Return (IRR)

Simple ROI = Total P&L ÷ gross deposits. One cumulative number; it ignores when you invested — a euro deployed on day 1 and a euro deployed yesterday count equally.

Money-Weighted Return (XIRR) = the single annualised rate r that makes the present value of every dated cash flow (each buy negative, each sell positive, today's value positive) sum to zero. It rewards deploying capital early and is the honest "what return did my actual money earn per year."

They answer different questions, so they don't match — and neither is the time-weighted return a fund reports (TWR strips out your deposit timing to judge the strategy; that needs a clean daily equity series, see below).

The accounting identity (why the numbers tie out)

Gross deposits − cost basis of shares sold = net cost basis (open). Net cost basis + unrealized P&L = current value. Realized + unrealized = total P&L. Cash returned from a sale = that share's cost basis + its realized gain — which is why net invested cash sits below net cost basis by exactly the realized profit you've banked.

Daily equity logging (for exact YTD / calendar returns)

Today YTD is reconstructed: holdings are repriced on every past business day from yfinance history. Correct, but recomputed each run and vulnerable to data revisions/splits. The robust upgrade is an append-only daily equity log (local/equity_log.csv, written every build): date, value, net cost basis, deposits, withdrawals, realized/unrealized. With it, calendar-year return becomes a flow-adjusted TWR — chain daily returns rᵢ = (Vᵢ − netflowᵢ)/Vᵢ₋₁ and compound across the year, so mid-year deposits never masquerade as performance. A daily cron/launchd job appending one row makes the series authoritative instead of reconstructed.

Risk metrics
  • Max drawdown — worst peak-to-trough fall of the value curve.
  • VaR 95% — daily loss not exceeded on 95% of days. CVaR 95% — average loss on the worst 5% of days.
  • Beta — sensitivity to the S&P 500. Alpha — return beyond what beta predicts. Sortino — Sharpe counting only downside.
